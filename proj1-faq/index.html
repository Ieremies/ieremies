1. Dicas em geral

Primeiro defina a estratégia, depois code. A maior parte do tempo não estamos escrevendo código, e sim pensando e tentando entender o que precisamos resolver e como resolveremos. Assim, tenha escrito as suas estratégias antes de pensar em codar. Jogue o joguinho e pense “nesse caso, qual seria a melhor estratégia aqui?” ou “se a resposta for (x, y), o que eu poderia fazer?”. Se você não tiver isso definido antes de codar, seu código nunca vai sair do lugar. Escreva no papel!

Evite utilizar variáveis globais. Variáveis globais são aquelas que vivem fora de todas as funções, como a lista colors que coloquei. Estas variáveis irão viver entre diferentes chamadas da sua função player, inclusive entre os jogos em si. Caso ainda sim você queira muito, lembre-se de “resetá-las” ao valor padrão quando o jogo recomeça, utilizando algo como: 

def player(guess_hist, res_hist):
    global variavel_global  # Indica se refere à variável global
    if len(guess_hist) == 0:
        lista_global = []
        variavel_global = 0

Observe que foi necessário dizer que a variavel_global a ser utilizada na função era, de fato, a global. Caso contrário, o python cria uma cópia dela que vive apenas dentro desta chamada. Isso não ocorre para listas, umas vez que elas são objetos mutáveis.

2. Dúvidas comuns

Como eu jogo o jogo sem rodar a minha função? Utilize o comando python3 src/main.py -t -i e colocar as letras iniciais das cores que você quer chutar. Perceba que cores como verde e vermelho precisam de no mínimo 4 letras para se diferenciarem.’

Como eu rodo o jogo uma vez utilizando a minha função para resolver? O mesmo comando acima, mas sem o -i ao final

Como eu testo o meu código 10.000 vezes? Utilize o comando python3 src/tournament.py.

Estou tendo o erro “can’t open file ‘src/tournament.py’: [Errno 2] No such file or directory”. Provavelmente você está tentando rodar os comandos acima já dentro da pasta src. Tente os comandos acima, mas sem a parte do src/

Estou tendo o erro “ModuleNotFoundError: No module named ‘pygame’” Isso acontece quando você está tentando utilizar o modo visual. Ou você instala a biblioteca pygame pelo comando pip3 install pygame ou não utiliza-a passando sempre a flag -t quando invocar o main.py para jogar no terminal. O tournament.py, por padrão, já não usa o modo visual.

Estou tendo o erro “match … SyntaxError: invalid syntax”. A sintaxe que utilizei só está disponível no python 3.10 ou superior. Confira a versão do seu python rodando, num terminal, o comando python3 --version. Caso a versão seja inferior a 3.10, procure como atualizar o seu sistema. No Linux, isso irá depender da sua distribuição, a melhor forma de descobrir é dar um google update python linux SUA_DISTRO ou perguntar a um monitor. Caso esteja no Windows, vá para o Linux.’

O que são os parâmetros da função player? Os parâmetros guess_hist e res_hist já são listas. O primeiro, é uma lista de palpites. Um palpite é uma lista de 4 cores. Ou seja, guess_hist[-1] é o último palpite e guess_hist[1][2] é a terceira cor do segundo palpite. Já res_hist é uma lista de tuplas, cada uma contendo dois valores: a quantidade de cores corretas e a quantidade de cores na posição correta. Tuplas são listas especiais em python, mas para todos os efeitos deste projeto podem ser tratadas como listas normais. Ou seja, res_hist[-1][0] é a quantidade de cores corretas do último palpite e res_hist[2][1] é a quantidade de cores na posição correta do terceiro palpite.

Como pegar o resultado de um palpite? Você tem duas opções: ou faz um contador para saber o índice do palpite: 

for i in range(len(guess_hist):

ou você pode “juntar” as duas listas usando a função do python chamada zip(). 

for guess, ress in zip(guess_hist, res_hist):

Neste último caso, ele junta as duas listas de forma que a variável res é o resultado do palpite guess e isso é feito na ordem do chute.
